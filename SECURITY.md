# Security

Do not open public issues that contain vulnerability details, API keys, tokens, private keys, certificates, provisioning profiles, Firebase config files, or production data.

If you find a security issue, report it through a private maintainer channel. If any secret or signing asset is accidentally exposed, treat it as compromised and rotate it before continuing development.

Real local configuration files are intentionally ignored by Git. Use the templates in this repository as references and keep live values outside the public tree.
